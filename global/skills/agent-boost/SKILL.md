# Agent Boost

Use `/agent-boost` when the user wants the setup, workflow, slash commands, or agent behavior to be more capable.

## Capability Upgrade Loop

1. Inventory current rules, workflows, skills, scripts, docs, and validation commands.
2. Identify the highest-friction tasks and convert them into slash workflows.
3. Tighten safety rules for destructive commands, secrets, package changes, and protected files.
4. Add stack-aware validation commands so the agent knows how to prove work is done.
5. Keep global guidance short; move specialized process into targeted skills or workflows.
6. Document exact usage examples so the next session can start fast.

## Commands

- `/agent-boost-audit`: Find missing rules, commands, checks, and automation.
- `/agent-boost-workflow`: Create a new slash workflow with inputs, steps, and exit criteria.
- `/agent-boost-validate`: Build a project-specific validation checklist.
