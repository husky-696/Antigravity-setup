# Premium Themes

Use `/themes` when the user asks for a nicer look, color palette, dark mode, light mode, brand vibe, or visual polish.

## Theme Modes

- **Midnight Glass**: zinc/black base, translucent cards, violet/cyan glow, soft radial gradients.
- **Aurora SaaS**: energetic gradient accents, confident CTAs, polished dashboard surfaces.
- **Editorial Light**: warm off-white backgrounds, crisp typography, restrained borders.
- **Brutalist Mono**: high-contrast black/white, oversized headings, minimal accents.
- **Warm Studio**: cream backgrounds, amber highlights, human-friendly surfaces.

## Workflow

1. Inspect existing tokens, Tailwind config, CSS variables, component library, and layout.
2. Define semantic tokens before editing components.
3. Apply tokens across app shell, cards, buttons, forms, focus rings, empty states, and charts.
4. Add tasteful motion and hover states without hurting accessibility.
5. Verify contrast, responsive behavior, and available build/lint checks.

## Commands

- `/themes-audit`: Review current UI and identify theme inconsistencies.
- `/themes-generate`: Create a semantic token set for a named vibe.
- `/theme`: Workspace slash workflow for applying a full theme upgrade.
