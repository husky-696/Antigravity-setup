# Elite Coding Standards

Use `/elite-code` to ensure your codebase follows industry-leading patterns and best practices.

## Core Principles
- **Clarity > Cleverness**: Write code that is easy to read and reason about.
- **DRY & KISS**: Don't repeat yourself, and keep it simple.
- **Type-Safety**: Enforce strict TypeScript types. Avoid `any` at all costs.
- **Architecture**: Use consistent patterns (e.g., Hooks for logic, Components for UI, Services for API).

## Refactoring Protocol
1. **Analyze**: Identify code smells (long functions, deep nesting, duplicate logic).
2. **Propose**: Suggest a cleaner architecture or pattern.
3. **Implement**: Perform the refactor with atomic, verified changes.
4. **Verify**: Ensure the refactor maintains functionality and improves performance.

## Commands
- `/elite-code-refactor`: Performs a deep refactor of a specific file or module.
- `/elite-code-audit`: Reviews the project for architectural inconsistencies and anti-patterns.
