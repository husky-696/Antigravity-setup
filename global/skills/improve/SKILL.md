# Feature & Design Evolution

Use `/improve` when you want to take existing code or UI to the next level. Focus on iterative polish, feature expansion, and premium refinements.

## Improvement Protocol
1. **Analyze Current State**: Identify the existing design/logic and its limitations.
2. **Propose Evolution**: Suggest specific improvements across three vectors:
   - **Visual (UI)**: Enhance aesthetics, spacing, and consistency.
   - **Experience (UX)**: Improve flow, feedback, and interaction speed.
   - **Functional (Logic)**: Add new capabilities or optimize existing ones.
3. **Elite Execution**: Implement the changes using the high-fidelity standards of the `/elite-*` skills.

## Focus Areas
- **UI Polish**: Applying glassmorphism, high-end typography, and vibrant palettes.
- **Micro-interactions**: Adding animations and haptic feedback.
- **Logic Hardening**: Improving error handling, state management, and performance.

## Commands
- `/improve-ui`: Focuses purely on visual and motion polish.
- `/improve-feat`: Focuses on adding or extending functionality.
- `/improve-ux`: Focuses on usability, speed, and interaction flow.
